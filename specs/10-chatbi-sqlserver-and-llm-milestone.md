# ChatBI SQL Server and LLM Milestone

## Purpose

This milestone combines two implementation steps into one delivery milestone while keeping two separate acceptance commits:

1. SQL Server dummy reporting layer.
2. DeepSeek V4 Flash intent parser.

The goal is to move ChatBI from a CSV-only backend tool executor to a database-backed and natural-language-testable dummy workflow.

## Scope

The milestone remains ChatBI-only.

It may add:

- SQL Server dummy staging and reporting objects for the approved PO-item tools.
- A backend repository that reads from the SQL Server reporting view.
- Environment-controlled switching between dummy CSV and SQL Server repositories.
- A DeepSeek-backed intent parser that turns a user question into one approved ChatBI tool call.
- A `/api/chatbi/ask` endpoint that validates the LLM output before tool execution.

It must not add:

- real SAP, PowerBI, supplier, price, purchasing, project, or internal report data;
- Supplier Knowledge Base tables or joins;
- frontend UI;
- free-form SQL generated by an LLM;
- raw database rows or query results in an LLM prompt.

## Commit A Boundary: SQL Server Dummy Reporting

Commit A creates a SQL Server dummy reporting path for the same synthetic PO-item fixture already used by tests.

Approved SQL objects:

- `chatbi.import_batch`
- `chatbi.po_item_staging`
- `chatbi.vw_po_item_reporting`

The SQL view is the first app-facing SQL reporting surface. It exposes approved PO-item fields in snake_case form. The backend SQL repository maps those fields back into the same internal row shape used by the CSV repository so the four BI tools do not change public contracts.

`DATA_MODE` controls repository selection:

- `dummy_csv`: default local mode;
- `sqlserver`: SQL Server reporting view mode.

SQL integration tests are opt-in. The normal test suite must pass without a local SQL Server connection.

## Commit B Boundary: DeepSeek V4 Flash Intent Parser

Commit B adds a provider abstraction for LLM intent parsing. The first provider is DeepSeek using its OpenAI-compatible chat-completions API.

Environment variables:

- `CHATBI_LLM_ENABLED=false` by default;
- `DEEPSEEK_API_KEY`;
- `DEEPSEEK_BASE_URL=https://api.deepseek.com`;
- `DEEPSEEK_MODEL=deepseek-v4-flash`.

The LLM receives only:

- the original user question;
- approved tool names and schemas;
- field dictionary summaries;
- scope and safety instructions.

The LLM must not receive raw rows, query results, database credentials, or SQL table names.

The LLM response must parse into an approved tool call. Backend validation remains authoritative.

`understood_request` remains backward-compatible for direct tool execution. The `/api/chatbi/ask` response adds:

- `intent_summary`: provider/model-scoped LLM parsing metadata;
- `execution_summary`: deterministic backend execution summary.

## Acceptance

The milestone is accepted when:

- default `python -m pytest -q` passes without SQL Server or DeepSeek credentials;
- SQL Server live tests are skipped unless explicitly enabled;
- DeepSeek live tests are skipped unless explicitly enabled;
- mocked tests prove prompt safety and tool-call validation;
- SQL-backed repository results match the CSV-backed repository contract;
- the two commits are independently reviewable and Commit A does not depend on LLM code.
