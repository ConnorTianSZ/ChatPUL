# ChatBI BI Tool JSON Schemas

## Purpose

This spec defines the first ChatBI BI tool JSON contracts.

These schemas describe the structured JSON that the LLM may produce when selecting a BI tool. They are not SQL schemas, not report lineage documents, and not backend implementation code.

The backend must validate the selected tool and arguments before execution. The LLM must not write SQL, request raw database rows, or add unapproved fields.

## Approved Tool Set

The first ChatBI version has four BI tools. Their names must stay aligned with the MVP data areas in `specs/04-chatbi-domain-spec.md`:

1. `supplier_dimension_summary`
2. `manufacturer_dimension_summary`
3. `lead_time_summary`
4. `auto_po_ratio_summary`

They are designed around the PO-item fact grain defined in `specs/04-chatbi-domain-spec.md`.

| MVP data area | Tool | Required grouping views |
| --- | --- | --- |
| Supplier dimension data | `supplier_dimension_summary` | supplier, buyer, manufacturer |
| Manufacturer dimension data | `manufacturer_dimension_summary` | manufacturer, buyer |
| Lead time data | `lead_time_summary` | buyer, manufacturer |
| Auto PO ratio data | `auto_po_ratio_summary` | buyer, manufacturer |

Supplier and manufacturer dimensions in these tools are ChatBI analytical dimensions only. They are not Supplier Knowledge Base supplier profiles, represented-manufacturer records, authorization records, or capability records.

## Shared Rules

Every tool call uses this envelope shape:

```json
{
  "tool": "tool_name",
  "arguments": {}
}
```

Shared validation rules:

- `tool` must match one of the approved tool names.
- `arguments` must match the selected tool schema.
- Unknown properties are rejected.
- Raw SQL, database names, table names, row payloads, and free-form formulas are not accepted as tool arguments.
- Date strings use `YYYY-MM-DD`.
- `time_range` defaults to `Doc. Date` / `doc_date` if omitted.
- Blank manufacturer handling is explicit. When grouped by manufacturer, blank source values should be returned under the label `<BLANK>` unless the user explicitly filters them out.
- MVP working-day calculations use Excel `NETWORKDAYS` semantics: Monday through Friday only, no public-holiday or company-calendar adjustment.

## Common Filter Fields

The schemas repeat these common filters directly so each tool contract can stand alone:

| Filter | Source meaning |
| --- | --- |
| `buyer_keys` | `PGr` values. |
| `supplier_codes` | `Vendor` values. |
| `supplier_names` | `Name 1` values. |
| `manufacturer_names` | `Manufactur` values, excluding blank unless `<BLANK>` is explicitly requested. |
| `include_blank_manufacturer` | Whether blank `Manufactur` rows are included. Defaults to true. |
| `wbs_scope` | `all`, `with_wbs`, or `blank_wbs`. |
| `plants` | `Plant` values. |
| `mrp_types` | `MRP Type` values. |

## Tool 1: `supplier_dimension_summary`

### Intent

Summarize supplier dimension data from the approved ChatBI PO-item reporting source.

The MVP stable measure is PO item count. Supplier dimension output may include supplier code, supplier name, buyer key, manufacturer grouping, WBS status, plant, MRP type, and reporting month when these fields are present in the approved source.

This tool does not query Supplier Knowledge Base profiles and does not infer supplier capability.

### Input JSON Schema

```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "title": "supplier_dimension_summary tool call",
  "type": "object",
  "required": ["tool", "arguments"],
  "additionalProperties": false,
  "properties": {
    "tool": { "const": "supplier_dimension_summary" },
    "arguments": {
      "type": "object",
      "required": ["group_by"],
      "additionalProperties": false,
      "properties": {
        "group_by": {
          "type": "string",
          "enum": ["supplier", "buyer", "manufacturer", "wbs_status", "plant", "mrp_type", "month"]
        },
        "time_range": {
          "type": "object",
          "additionalProperties": false,
          "properties": {
            "date_field": {
              "type": "string",
              "enum": ["doc_date", "pr_date", "order_confirmation_date", "goods_receipt_posting_date", "statistical_delivery_date"],
              "default": "doc_date"
            },
            "from": { "type": "string", "pattern": "^\\d{4}-\\d{2}-\\d{2}$" },
            "to": { "type": "string", "pattern": "^\\d{4}-\\d{2}-\\d{2}$" }
          }
        },
        "filters": {
          "type": "object",
          "additionalProperties": false,
          "properties": {
            "buyer_keys": { "type": "array", "items": { "type": "string" } },
            "supplier_codes": { "type": "array", "items": { "type": "string" } },
            "supplier_names": { "type": "array", "items": { "type": "string" } },
            "manufacturer_names": { "type": "array", "items": { "type": "string" } },
            "include_blank_manufacturer": { "type": "boolean", "default": true },
            "wbs_scope": { "type": "string", "enum": ["all", "with_wbs", "blank_wbs"], "default": "all" },
            "plants": { "type": "array", "items": { "type": "string" } },
            "mrp_types": { "type": "array", "items": { "type": "string" } }
          }
        },
        "sort": {
          "type": "object",
          "additionalProperties": false,
          "properties": {
            "by": { "type": "string", "enum": ["po_item_count", "group_key"] },
            "direction": { "type": "string", "enum": ["asc", "desc"], "default": "desc" }
          }
        },
        "limit": { "type": "integer", "minimum": 1, "maximum": 100 }
      }
    }
  }
}
```

### Result Contract

Expected result fields:

- `tool`: `supplier_dimension_summary`;
- `understood_request`: one sentence;
- `group_by`;
- `filters_applied`;
- `rows`: grouped result rows with `group_key`, `po_item_count`, and optional supplier code/name fields when `group_by = "supplier"`;
- `total`: total PO item count after filters;
- `source_trace`.

## Tool 3: `lead_time_summary`

### Intent

Summarize PR lead time and PO-confirmation lead time.

PR lead time formula:

`NETWORKDAYS(PR Date, Doc. Date) - 1`

PO-confirmation lead time formula:

`NETWORKDAYS(Doc. Date, order confirmatioin date) - 1`

Rows with missing or invalid `PR Date` are classified as `not statistic`. Rows with blank order confirmation dates are classified as `Delivered without OC` when `GR-D.o.Post` is populated, otherwise `no order confirmation`.

### Input JSON Schema

```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "title": "lead_time_summary tool call",
  "type": "object",
  "required": ["tool", "arguments"],
  "additionalProperties": false,
  "properties": {
    "tool": { "const": "lead_time_summary" },
    "arguments": {
      "type": "object",
      "required": ["lead_time_type", "group_by"],
      "additionalProperties": false,
      "properties": {
        "lead_time_type": {
          "type": "string",
          "enum": ["pr", "po_confirmation", "both"]
        },
        "group_by": {
          "type": "string",
          "enum": ["none", "buyer", "manufacturer", "supplier", "month"]
        },
        "time_range": {
          "type": "object",
          "additionalProperties": false,
          "properties": {
            "date_field": {
              "type": "string",
              "enum": ["doc_date", "pr_date", "order_confirmation_date", "goods_receipt_posting_date"],
              "default": "doc_date"
            },
            "from": { "type": "string", "pattern": "^\\d{4}-\\d{2}-\\d{2}$" },
            "to": { "type": "string", "pattern": "^\\d{4}-\\d{2}-\\d{2}$" }
          }
        },
        "filters": {
          "type": "object",
          "additionalProperties": false,
          "properties": {
            "buyer_keys": { "type": "array", "items": { "type": "string" } },
            "supplier_codes": { "type": "array", "items": { "type": "string" } },
            "supplier_names": { "type": "array", "items": { "type": "string" } },
            "manufacturer_names": { "type": "array", "items": { "type": "string" } },
            "include_blank_manufacturer": { "type": "boolean", "default": true },
            "wbs_scope": { "type": "string", "enum": ["all", "with_wbs", "blank_wbs"], "default": "all" },
            "plants": { "type": "array", "items": { "type": "string" } },
            "mrp_types": { "type": "array", "items": { "type": "string" } }
          }
        }
      }
    }
  }
}
```

### Result Contract

Expected result fields:

- `tool`: `lead_time_summary`;
- `understood_request`: one sentence;
- `lead_time_type`;
- `group_by`;
- `pr_buckets` when `lead_time_type` is `pr` or `both`, with counts for `0<=X<=3`, `3<X<=7`, `>7days`, and `not statistic`;
- `po_confirmation_buckets` when `lead_time_type` is `po_confirmation` or `both`, with counts for `<=3`, `3<X<=7`, `>7`, `no order confirmation`, and `Delivered without OC`;
- `groups`: optional grouped bucket counts;
- `calculation_note`;
- `source_trace`.

## Tool 4: `auto_po_ratio_summary`

### Intent

Calculate automatic PO ratio from PO item rows.

Definition:

`count(PO created by = UC4CPIC) / count(all PO item rows in the same filter context)`

The numerator rule is fixed for the MVP. Tool arguments may filter the dataset, but they must not redefine the numerator creator.

### Input JSON Schema

```json

  "properties": {

    "arguments": {
      "type": "object",
      "required": ["group_by"],
      "additionalProperties": false,
      "properties": {
        "group_by": {
          "type": "string",
          "enum": ["none", "buyer", "manufacturer", "supplier", "month"]
        },
        "numerator_rule": {
          "type": "string",
          "const": "po_created_by_equals_uc4cpic"
        },
        "time_range": {
          "type": "object",
          "additionalProperties": false,
          "properties": {
            "date_field": {
              "type": "string",
              "enum": ["doc_date", "pr_date", "order_confirmation_date", "goods_receipt_posting_date", "statistical_delivery_date"],
              "default": "doc_date"
            },
            "from": { "type": "string", "pattern": "^\\d{4}-\\d{2}-\\d{2}$" },
```

- `overall` or `groups`, each with `uc4_count`, `total`, and `ratio`;
- `source_trace`.

## Tool Selection Guidance

The LLM should select tools by business intent:

| User intent | Tool |
| --- | --- |
| Supplier list, supplier PO item count, supplier coverage, supplier split by buyer or manufacturer | `supplier_dimension_summary` |
| Manufacturer list, manufacturer PO item count, blank manufacturer bucket, manufacturer split by buyer | `manufacturer_dimension_summary` |
| PR lead time, PO confirmation lead time, missing OC, delivered without OC | `lead_time_summary` |
| Auto PO ratio, UC4 share, automation rate | `auto_po_ratio_summary` |

If a user asks for unsupported metrics such as price trend, value-weighted automation ratio, quality risk, or delivery risk, the backend should reject the tool call or ask for clarification until a later approved schema exists.
